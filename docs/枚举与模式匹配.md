# Rust 技术专题：枚举（Enum）与模式匹配详解

枚举（enum）是 Rust 中最强大的复合类型之一，它允许你通过列举所有可能的**变体（variants）**来定义一个类型。与 C/C++ 中仅能表示整数的枚举不同，Rust 的枚举变体可以**携带任意类型的数据**，是真正的"带标签的联合体"。

---

## 一、枚举的定义与基本用法

### 1. 最简单的枚举

```rust
enum Direction {
    Up,
    Down,
    Left,
    Right,
}

let d = Direction::Up;
```

每个变体都是 `Direction` 类型的一个可能值，通过 `::` 访问。

### 2. 携带数据的枚举变体

Rust 枚举的强大之处在于每个变体可以关联不同类型和数量的数据：

```rust
enum Message {
    Quit,                       // 单元变体（无数据）
    Move { x: i32, y: i32 },   // 结构体变体（命名字段）
    Write(String),              // 元组变体（一个 String）
    ChangeColor(i32, i32, i32), // 元组变体（三个 i32）
}
```

上面定义等价于定义了四个不同的结构体，但它们被统一在 `Message` 这一个类型下，可以用同一个函数来处理所有情况。

### 3. 创建枚举实例

```rust
let quit = Message::Quit;
let mv = Message::Move { x: 10, y: 20 };
let write = Message::Write(String::from("hello"));
let color = Message::ChangeColor(255, 0, 0);
```

---

## 二、为枚举实现方法

和结构体一样，可以使用 `impl` 为枚举定义方法：

```rust
impl Message {
    fn call(&self) {
        match self {
            Message::Quit => println!("退出"),
            Message::Move { x, y } => println!("移动到 ({}, {})", x, y),
            Message::Write(text) => println!("写入: {}", text),
            Message::ChangeColor(r, g, b) => println!("颜色: ({}, {}, {})", r, g, b),
        }
    }
}

let m = Message::Write(String::from("hello"));
m.call(); // 输出: 写入: hello
```

## 三、match 模式匹配

`match` 是 Rust 处理枚举的核心控制流结构，类似于其他语言的 `switch`，但功能更强大。

### 1. 基本语法

```rust
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u32 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}
```

### 2. 核心规则：穷尽性（Exhaustiveness）

`match` 必须覆盖枚举的**所有变体**，否则编译器报错。这是 Rust 安全性的重要保证。

```rust
// 编译错误！缺少 Coin::Quarter 分支
match coin {
    Coin::Penny => 1,
    Coin::Nickel => 5,
    Coin::Dime => 10,
    // 缺少 Quarter 会导致编译失败
}
```

### 3. 通配模式与 `_` 占位符

当不需要处理所有变体时，可以用 `other` 或 `_` 捕获剩余情况：

```rust
let dice_roll = 9;
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    other => move_player(other), // other 绑定了值
}

// 或使用 _ 忽略值
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    _ => (), // 什么也不做
}
```

### 4. 绑定变体中的数据

```rust
enum Action {
    Say(String),
    MoveTo(i32, i32),
    Stop,
}

fn perform(action: Action) {
    match action {
        Action::Say(msg) => println!("说: {}", msg),
        Action::MoveTo(x, y) => println!("移动到 ({}, {})", x, y),
        Action::Stop => println!("停止"),
    }
}
```

### 5. match 守卫（Match Guard）

在匹配分支中添加额外条件：

```rust
let num = Some(4);
match num {
    Some(x) if x < 5 => println!("小于5: {}", x),
    Some(x) => println!("其他值: {}", x),
    None => println!("无值"),
}
```

## 四、if let 与 while let 语法糖

### 1. if let — 简化单分支匹配

当只关心一个变体时，`match` 显得冗长。`if let` 提供了更简洁的写法：

```rust
let some_value = Some(3);

// 使用 match（冗长）
match some_value {
    Some(3) => println!("是三！"),
    _ => (),
}

// 使用 if let（简洁）
if let Some(3) = some_value {
    println!("是三！");
}
```

#### 配合 else 使用

```rust
let coin = Coin::Penny;

if let Coin::Quarter = coin {
    println!("是25美分！");
} else {
    println!("不是25美分");
}
```

### 2. while let — 循环匹配

只要模式持续匹配，循环就会继续执行：

```rust
let mut stack = Vec::new();
stack.push(1);
stack.push(2);
stack.push(3);

// pop() 返回 Option<T>，当返回 None 时循环结束
while let Some(top) = stack.pop() {
    println!("{}", top);
}
// 输出: 3 2 1
```

### 3. let else — 提前返回（Rust 1.65+）

```rust
fn get_count(text: &str) -> Result<usize, String> {
    let Ok(count) = text.parse::<usize>() else {
        return Err(format!("无法解析: {}", text));
    };
    Ok(count)
}
```

## 五、Option 枚举 — 处理值的缺失

Rust 没有 `null`，取而代之的是标准库中的 `Option<T>` 枚举：

```rust
enum Option<T> {
    None,
    Some(T),
}
```

### 1. 基本用法

```rust
let some_number: Option<i32> = Some(5);
let no_number: Option<i32> = None;

// Option<T> 和 T 是不同的类型，不能直接运算
let x: i32 = 5;
let y: Option<i32> = Some(5);
// let sum = x + y; // 编译错误！
```

### 2. 提取 Option 中的值

```rust
let val = Some(42);

// 方式一：match
match val {
    Some(v) => println!("值是: {}", v),
    None => println!("没有值"),
}

// 方式二：if let
if let Some(v) = val {
    println!("值是: {}", v);
}

// 方式三：unwrap（None 时 panic，慎用）
let v = val.unwrap();

// 方式四：unwrap_or（提供默认值）
let v = val.unwrap_or(0);

// 方式五：unwrap_or_else（惰性计算默认值）
let v = val.unwrap_or_else(|| compute_default());

// 方式六：map（对 Some 中的值进行转换）
let doubled = val.map(|v| v * 2); // Some(84)
```

### 3. Option 的常用方法

| 方法                   | 说明                                     |
| ---------------------- | ---------------------------------------- |
| `is_some()`            | 如果是 `Some` 返回 `true`               |
| `is_none()`            | 如果是 `None` 返回 `true`               |
| `unwrap()`             | 取出值，`None` 时 panic                  |
| `unwrap_or(default)`   | 取出值或返回默认值                       |
| `unwrap_or_else(f)`    | 取出值或调用闭包计算默认值               |
| `map(f)`               | 对 `Some(v)` 应用函数 `f`，返回新 Option |
| `and_then(f)`          | 链式操作，`f` 返回 `Option`              |
| `or(other)`            | 如果是 `None`，返回 `other`              |
| `filter(predicate)`    | 如果谓词为 `false`，变为 `None`          |

## 六、Result 枚举 — 错误处理

`Result<T, E>` 用于表示可能失败的操作：

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

### 1. 基本用法

```rust
use std::fs::File;

fn main() {
    let f = File::open("hello.txt");

    let file = match f {
        Ok(file) => file,
        Err(error) => panic!("打开文件失败: {:?}", error),
    };
}
```

### 2. `?` 运算符 — 错误传播的语法糖

```rust
use std::fs;
use std::io;

fn read_username() -> Result<String, io::Error> {
    let mut s = String::new();
    File::open("hello.txt")?.read_to_string(&mut s)?;
    Ok(s)
}

// 更简洁的写法
fn read_username_short() -> Result<String, io::Error> {
    fs::read_to_string("hello.txt")
}
```

`?` 的作用：如果 `Result` 是 `Ok`，取出其中的值继续执行；如果是 `Err`，提前返回错误。

### 3. Result 的常用方法

| 方法                  | 说明                                    |
| --------------------- | --------------------------------------- |
| `is_ok()`             | 是否为 `Ok`                             |
| `is_err()`            | 是否为 `Err`                            |
| `unwrap()`            | 取出 `Ok` 的值，`Err` 时 panic          |
| `expect(msg)`         | 类似 `unwrap`，但 panic 时显示自定义信息 |
| `unwrap_or(default)`  | 取出值或返回默认值                      |
| `map(f)`              | 对 `Ok(v)` 应用函数                     |
| `map_err(f)`          | 对 `Err(e)` 应用函数                    |
| `and_then(f)`         | 链式操作                                |
| `ok()`                | 转换为 `Option<T>`                      |

## 七、高级用法

### 1. 设置判别值（Discriminant）

可以为枚举变体手动指定整数值：

```rust
enum HttpStatus {
    Ok = 200,
    NotFound = 404,
    InternalServerError = 500,
}

let status = HttpStatus::Ok as i32; // 200
```

### 2. C 风格枚举（类C枚举）

```rust
#[repr(u8)]  // 指定底层存储类型
enum Color {
    Red = 0,
    Green = 1,
    Blue = 2,
}

// 可以转为整数
let r = Color::Red as u8; // 0
```

### 3. 枚举的内存布局

- 枚举的大小等于**最大变体的大小** + **判别式的大小**
- 编译器会进行**小众优化（Niche Optimization）**：例如 `Option<&T>` 的大小与 `&T` 相同，因为引用不可能为 null，编译器用 null 值表示 `None`

```rust
use std::mem::size_of;

// &T 是 8 字节（64位系统），Option<&T> 也是 8 字节！
assert_eq!(size_of::<&u8>(), size_of::<Option<&u8>>());
```

### 4. 递归枚举

枚举可以引用自身，但必须使用间接引用（如 `Box`）：

```rust
// 链表
enum List {
    Cons(i32, Box<List>),
    Nil,
}

use List::{Cons, Nil};

let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
```

### 5. 泛型枚举

```rust
enum Tree<T> {
    Leaf(T),
    Node(Box<Tree<T>>, Box<Tree<T>>),
}

let tree = Tree::Node(
    Box::new(Tree::Leaf(1)),
    Box::new(Tree::Leaf(2)),
);
```

### 6. 使用 use 简化枚举访问

```rust
enum Status {
    Rich,
    Poor,
}

// 通过 use 将变体导入作用域
use Status::{Rich, Poor};
// 或导入所有变体
// use Status::*;

let status = Poor; // 无需写 Status::Poor
```

## 八、常见模式与最佳实践

### 1. 状态机模式

```rust
enum ConnectionState {
    Disconnected,
    Connecting(String),     // 存储连接地址
    Connected(TcpStream),   // 持有连接
    Error(String),          // 存储错误信息
}

fn handle(state: ConnectionState) -> ConnectionState {
    match state {
        ConnectionState::Disconnected => {
            println!("正在连接...");
            ConnectionState::Connecting("127.0.0.1".to_string())
        }
        ConnectionState::Connecting(addr) => {
            println!("已连接到 {}", addr);
            // 假设连接成功
            ConnectionState::Connected(/* ... */)
        }
        // ...
    }
}
```

### 2. 命令模式

```rust
enum Command {
    Create { name: String },
    Delete { id: u64 },
    Update { id: u64, name: String },
    List,
}

fn execute(cmd: Command) {
    match cmd {
        Command::Create { name } => println!("创建: {}", name),
        Command::Delete { id } => println!("删除: {}", id),
        Command::Update { id, name } => println!("更新 {}: {}", id, name),
        Command::List => println!("列出所有"),
    }
}
```

### 3. 常用 derive 宏

```rust
#[derive(Debug, Clone, PartialEq)]
enum Shape {
    Circle(f64),
    Rectangle(f64, f64),
    Triangle(f64, f64, f64),
}

let s = Shape::Circle(5.0);
println!("{:?}", s); // 输出: Circle(5.0)
```

## 九、总结对比

| 特性       | 结构体（Struct）       | 枚举（Enum）             |
| ---------- | ---------------------- | ------------------------ |
| 用途       | 将多个字段组合为一个类型 | 表示"多选一"的类型       |
| 数据       | 所有字段同时存在         | 同一时刻只有一个变体有效 |
| 内存       | 所有字段大小之和         | 最大变体大小 + 判别式    |
| 模式匹配   | 解构                    | `match` 穷尽匹配         |
| 典型场景   | 数据记录、配置项        | 状态、命令、错误处理     |
